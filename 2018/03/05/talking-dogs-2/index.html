Something we’re considering including in our project is the use of Google’s translation services in order to expand our app’s accessibility, language-wise. Part of the appeal of it is just that it’s so simple to use - just put something in and it spits out the result in a very clear, easy-to-understand manner.

Very easy. What’s important to understand about this translation and how to use it is, however, two things: firstly, that to be able to use it effectively, the user needs to know the ISO 639-1 language code, as well as the two different translation models (not specified in this particular sample code, but important) that Google’s API uses for its translations.

The ISO 639-1 language code is pretty self explanatory; to be able for the API to understand what langauge it should be translating to (and from, if the specificity is important), the user must provide this. Wikipedia has a handy list of language codes.

The translation models are a bit more complicated and different translations will be produced depending on which one is used. There are two to know, Phrase-Based Machine Translation (PBMT) and Neural Machine Translation (NMT).

Phrase-Based Machine Translation

is a type of statistical machine translation where it guesses the probable translation for sets of phrases based on a model created from parallel, pre-translated texts; it translates based on the likelihood of a particular meaning in that context, obtained from the massive amounts of data used to build the model. This is the model traditionally used in machine translation, developed and studied over the past several decades and, perhaps to the chagrin of pretty much anyone who’s tried using a machine translator, typically results in higly literal translations that don’t often make a lot of sense. It has been suggested, however, that PBMT functions well when dealing with highly technical texts, where other aspects of language (context, tone, etc) can be safely ignored.

Neural Machine Translation

is a much more recent, emerging technology that Google has started using for its translations - when the translation model is not specified, such as in the sample code above, Google will by default attempt to translate using NMT. It is modelled after neural networks in the human brain using deep learning techniques in order to attempt to emulate the way humans learn, and uses algorithms to teach itself language rules. Because of this, NMT typically provides higher-quality translations at a much faster speed compared to PBMT.

(For more reading: Matecat.com: General - PBMT vs NMT, United Language Group: Statistical Vs. Neural Machine Translation)

In order to specify which model is to be used when translating, model: nmt or model: base (for PBMT) needs to be added to the request body.

But which one holds up better, really?

Here’s a piece of sample text from a Wikipedia article:

ギャルは、英語において若い女性を指す girl の、アメリカ英語における俗語 gal に由来する外来語。 日本語でも英語と同様の意味を持つ場合があるが、とりわけファッションやライフスタイルが突飛と見なされながらも、それらが同世代にある程度文化として共有されている若い女性たちを指す場合にも用いる。

A natural (albeit unedited) translation made by a human translator might be,

Gyaru is a loanword of the word ‘gal’ (slang used in American-English), derived from the English word ‘girl’, which is used to indicate a young woman. Although there are situations wherein the meaning of the word in both English and Japanese contexts are the same, it typically indicates a particular fashion or lifestyle, or young women who engage in this contemporary subculture.

(For more reading: Wikipedia entry on Gyaru.)

A translation using PBMT:

Gal, the girl pointing to a young woman in English, foreign language derived from the slang gal in American English. There is a case to have the same meaning as English in Japanese, but especially also with fashion and lifestyle is regarded as outlandish, also used if they refer to the young women that are shared to some extent as culture in the same generation.

And a translation using NMT:

Girl is a foreign word derived from slang gal in American English, of girl pointing to a young woman in English. Although it may have the same meaning as English in Japanese, it is used also when young women who are referring to young women who are shared to some extent as a culture to the same generation, while fashion and lifestyle are considered to be particularly funny.

When we compare the PBMT and NMT translations, we can see that, while both are somewhat confusing to understand and have their own grammatical and lexical issues, PBMT is by far the more stilted of the two. PBMT selects words and phrases that are technically correct translations, but are infrequently used in regular vernacular. NMT attempts to replace these with more natural language.

However, in the case of CheerUpCharm, neither of these sample translations would be of much relevance - the app itself does not currently contain large amounts of text, and any text that is being used is either simple menu text, or dialogue from the dog/chatbot.

Another sample translation, this time with a more conversational tone (the sample text was retrieved from a post on Twitter):

そういえば、年明けあたりにソーシャルゲーム業界の人が「過去の統計から２月はソーシャルゲームの売上が軒並み落ちる事が判っています」って言ってて、なんか漁師みたいだなーと思った。

Natural translation:

Oh yeah, around New Years’ someone in the social-network game industry was saying that given past stats, we know February sales in social-network games are going to drop pretty steadily and that just made me think huh, kinda sounds like a fisherman.

PBMT:

Speaking of which, they say per turn of the year people of the social game industry is me "February from the past of Statistics has found that the sales of social games to fall across the board", I thought Na's like something fisherman.

NMT:

Speaking of which, people in the social gaming industry said that from the past statistics it is known that sales of social games are falling down in February and it seems that somehow seems like a fisherman.

In this case, the translation issues aside, the PBMT model attempted to use quotation marks to match the original Japanese input, where the NMT model removed them entirely - which is a common translation decision to make, as quotations in Japanese aren’t necessarily as strict as verbatim quotes like in English.

Either way, as far as translation accuracy and fidelity goes, neither are really suitable to use for conversational dialogue, so while it’s a nice idea and could maybe be used in a pinch, machine translation probably oughtn’t be used, at least not outside of some sort of beta/demo feature. It might be acceptable for single-word translations like a menu, though.
